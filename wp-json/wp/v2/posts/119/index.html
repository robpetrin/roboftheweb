For new and experienced web developers, assessing tools and resources is a critical part of your practice. This includes web references, web tools, code editors, CMS software, and more. In this article, we’re going to assess seven distinct tools, and why you should select them over their chief competitors.<\/p>\n\n\n\n

None of the applications below are sponsoring this post, or my recommendation of them. Feel free to do your own research — Googling is a developer’s best friend!<\/strong><\/p>\n\n\n\n

Instant Prototyping: CodePen<\/a><\/strong><\/h2>\n\n\n\n

Price: <\/strong>Free (with a Pro tier)<\/p>\n\n\n\n

I use CodePen almost daily when creating small prototypes, one-off WordPress components, or mini-projects to showcase. It includes editor panes for HTML, CSS, and JavaScript, and supports processors and supersets of each language. It’s an excellent sandbox to try out code, fork someone else’s Pen, or to experiment with external libraries and frameworks.<\/p>\n\n\n\n

For free, you get unlimited public pens and unlimited templates. For Pro ($8\/mo), you get asset hosting (file storage), project support, collaboration and debug mode, private pens, ad removal, and live view.<\/p>\n\n\n\n

Chief Competitor: <\/strong>JSFiddle<\/a>. While CodePen is more intended for front-end mockups, JSFiddle is used primarily for JavaScript experimenting. You have to hit “Run” to see your updates, whereas this is instantaneous on CodePen. So, depending upon your use-case, JSFiddle might be a better or worse fit. For front-end purposes, typically CodePen is a sleeker editing experience.<\/p>\n\n\n\n

Code Editing: VSCode<\/a><\/strong><\/h2>\n\n\n\n

Price: <\/strong>Free (open-source)<\/p>\n\n\n\n

Visual Code Studio is my pick for two main reasons — it’s the most popular, and because of it’s extension ecosystem. Keep those factors in mind when I get to CMSes later in this article.<\/p>\n\n\n\n

Your IDE (integrated development environment) is the most important tool you’ll use when writing code, for obvious reasons. It needs to be crafted to your workflow — colors, plugins, presets, keyboard shortcuts, etc. — to ensure you continue to work quickly and productively. VSCode is highly configurable, and widely adopted.<\/p>\n\n\n\n

Chief Competitor: <\/strong>Atom<\/a>. There’s a lot to like with Atom, which is based on Electron (Node.JS). It’s been plagued by performance issues in the past, and has nowhere near the adoption of VSCode. But it’s worth a look.<\/p>\n\n\n\n

Design Mockups: Figma<\/a><\/strong><\/h2>\n\n\n\n

Price: <\/strong>Free (with paid tiers)<\/p>\n\n\n\n

This is assuming my audience is primary developers. For Designers, I might have a different recommendation. However, I love Figma because it feels like a design tool with developers in mind. Like VSCode, a wide plugin ecosystem doesn’t hurt. In fact, a number of plugins can help the export and initial development cycles. Such a time-saver.<\/p>\n\n\n\n

Chief Competitor: <\/strong>Adobe XD<\/a>. It’s recommend this to designers, as it’s a lovely tool. But it does more to scaffold web design for traditional designers than it does to scaffold web developer handover. However, it still does a good job of the latter the same way it does the former.<\/p>\n\n\n\n

Local Development: Local by Flywheel<\/a><\/strong><\/h2>\n\n\n\n

Price: <\/strong>Free (but hosting isn’t)<\/p>\n\n\n\n

A big reason for this recommendation is that this is owned by Flywheel (good), which is itself owned by WPEngine (great). The ability to, in a single click, sync from Local to remote and back is key. It’s incredibly easy to spin up a local WordPress instance in minutes, and this makes working with a three-environment setup a no-brainer.<\/p>\n\n\n\n

It is the simplest way to work with WordPress locally — full-stop.<\/p>\n\n\n\n

Chief Competitor: <\/strong>WAMP, MAMP, and LAMP<\/a> — but why go through the abstraction if it’s not key to your hosting circumstances? I really cannot recommend considering this alternative unless it fits your use-case — in which case you’re likely not a beginner or relatively new to development.<\/p>\n\n\n\n

Version Management: GitHub<\/a><\/strong><\/h2>\n\n\n\n

Price: <\/strong>Free (with paid tiers)<\/p>\n\n\n\n

There’s ethical reasons<\/a> to reconsider GitHub. However, its wide adoption means it is a big step down to one of its competitors. Ultimately, all Git providers are the same. It’s underlying technology that neither GitHub nor its competitors solely control. It’s just a brand-name thing — but keep an eye out for a wider boycott I suppose, which can hurt its market leader status.<\/p>\n\n\n\n

Chief Competitors: <\/strong>GitLab<\/a>, BitBucket<\/a> — truly, all Git providers are roughly the same.<\/p>\n\n\n\n

Content Management System: WordPress<\/a><\/strong><\/h2>\n\n\n\n

Price: <\/strong>Free (open-source)<\/p>\n\n\n\n

WordPress, as I foreshadowed, is my pick due to market share (~1\/3 of all websites run WordPress, and almost 2\/3 of sites using a CMS are using WordPress) and plugin ecosystem. For every website use case, there’s a WordPress solutions. In so many cases, it’s better than the alternative of building something natively.<\/p>\n\n\n\n

With the advent of Gutenberg and Full-Site Editing in 2022, the landscape for WordPress developers is going to change. Drastically. And quickly. While WordPress isn’t expected to remove PHP from it’s core code anytime soon, it’s neutered the need for any developers to touch it besides here-and-there in 2022. It’s possible to utilize all of WordPress without touching PHP. What a glorious sentence.<\/p>\n\n\n\n

Chief Competitors: <\/strong>Drupal<\/a>, Ghost<\/a>, Static Site Generators. Of these —<\/p>\n\n\n\n

  • Drupal is antiquated. If WordPress is Target, Drupal is Sears. There’s no reason to pick it in 2022, and anyone who has a reason other than “cybersecurity” are lying. Anyone who says “cybersecurity” has old information.<\/li>
  • Ghost is the closest experience I’ve seen to WordPress, but is a few feature releases behind it to be a real threat. It’s smooth, and definitely influenced by WordPress, and is written in Node.JS. However, it’s far less extendible than WordPress, and woefully under-featured to overcome that limitation with extension. Check back in five years.<\/li>
  • Static Sites are the newest fad, but they provide legitimate security. There’s nothing to hack if your site is headless\/decoupled from the data source, which could also still be a WordPress instance. These orchestrations are a little more work, but can help make your hosting cost as little as $0\/month if you pick the right host.<\/li><\/ul>\n\n\n\n

    Cloud Hosting: Netlify<\/a><\/strong><\/h2>\n\n\n\n

    Price: <\/strong>Free, but paid if your usage grows high enough<\/p>\n\n\n\n

    My favorite place to host static projects is on Netlify. It can do so much more than what I ask of it, and usually does it for free. While it cannot host WordPress, it easily hosts static instances of WordPress when paired with a static site-gen plugin such as Simply Static<\/a>.<\/p>\n\n\n\n

    Simplicity is the aim of Netlify, and it nails it. This will be the standard of hosting in five years, as it has its own CMS implementation (not a threat to WordPress, but remarkable). And you can’t beat free.<\/p>\n\n\n\n

    Chief Competitors: <\/strong>Digital Ocean<\/a>, WPEngine<\/a>, AWS<\/a> — none of these quite do what Netlify does. However, I can recommend each of these for slightly different use-cases.<\/p>\n\n\n\n

    Development Browser: Chrome<\/a><\/strong><\/h2>\n\n\n\n

    Price: <\/strong>Free<\/p>\n\n\n\n

    Much has been written about Firefox having a superior Inspector experience<\/a>, which is accurate. However, Google powers Chrome, is the market leader, and provides tools to optimize for performance, which the biggest search engine prioritizes in page rank — which just so happens to also be Google. For that reason alone, Chrome Lighthouse trumps Firefox Inspector (and Chrome’s Inspector is no slouch).<\/p>\n\n\n\n

    In a perfect world, you’re development in each browser to gain the benefits of each. In a regular world, you’re definitely doing cross-browser review in each of them. But if you can only pick one, I say Chrome. It has a slightly larger extension ecosystem, and is in your SEO best interest.<\/p>\n\n\n\n
